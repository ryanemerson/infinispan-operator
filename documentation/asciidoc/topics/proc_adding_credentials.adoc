[id='adding_credentials-{context}']
= Adding Custom User Credentials
Configure access to {brandname} cluster endpoints with custom credentials.

.Procedure

. Create an `identities.yaml` file with the credentials that you want to add.
+
[source,yaml,options="nowrap",subs=attributes+]
----
credentials:
- username: testuser
  password: testpassword
----
+
. Create an authentication secret from `identities.yaml`.
+
[source,options="nowrap",subs=attributes+]
----
$ {oc_create} secret generic --from-file=identities.yaml connect-secret
----
+
. Specify the authentication secret with `spec.security.endpointSecretName` in your `Infinispan` CR and then apply the changes.
+
[source,options="nowrap",subs=attributes+]
----
include::yaml/cr_auth.yaml[]
----
+
<1> Specifies the name of the authentication secret that contains your
credentials.

Modifying `spec.security.endpointSecretName` triggers a cluster restart. You
can watch the {brandname} cluster as {ispn_operator} applies changes:

[source,options="nowrap",subs=attributes+]
----
$ {oc_get_pods_w}
----

[id='changing__operator_password-{context}']
= Chaning the Operator Password
The operator automatically generates a password for the `operator` user account, however it's possible to update this value
if required.

.Procedure

. Update the `password` key in the `<infinispan-cr-name>-generated-operator-secret` secret to contain the desired password.

[source,bash,options="nowrap",subs=attributes+]
----
kubectl patch secret <infinispan-cr-name>-generated-operator-secret -p='{"stringData":{"password": "supersecretoperatorpassword"}}'
----

[NOTE]
====
The `<infinispan-cr-name>-generated-operator-secret` secret contains multiple keys, however it is only necessary to update
the `password` key. This will cause all other content to be automatically generated.
====
